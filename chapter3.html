3.1 Tabel: Mangrove_Conservation_Records.csv

Tabel utama untuk merekam proyek konservasi mangrove:

Variabel Tipe Data Deskripsi Contoh Nilai Kaitan Blockchain
Conservation_ID String (Primary Key) Kode unik proyek (format: CXXX) C001 Kunci utama untuk relasi antar tabel
Location String Lokasi proyek (kabupaten/kota) Aceh Jaya Untuk pemetaan geografis
Area_Ha Float Luas area konservasi (hektare) 50.0 Menghitung potensi karbon
Carbon_Credits Integer Jumlah kredit karbon (1 kredit = 1 ton CO₂) 250 Aset digital di blockchain
Date_Recorded Date Tanggal pencatatan data (YYYY-MM-DD) 2024-01-15 Timestamp transaksi

Contoh Data Lengkap:

Conservation_ID Location Area_Ha Carbon_Credits Date_Recorded
C001 Aceh Jaya 50.0 250 2024-01-15
C002 Banten 75.5 378 2024-02-20
C003 Kalimantan Barat 120.0 600 2024-03-10

Diagram Relasi:

Mangrove_Conservation_Records.csv → Blockchain_Transactions.csv (1-to-many)

Setiap proyek konservasi dapat memiliki banyak transaksi karbon

Validasi Data:

  • Conservation_ID: Harus unik, format C diikuti 3 digit
  • Area_Ha: Minimal 0.1 ha, maksimal 10,000 ha
  • Carbon_Credits: Harus ≤ Area_Ha × 20 (asumsi maks 20 ton/ha)

3.2 Tabel: Blockchain_Transactions.csv

Mencatat semua transaksi kredit karbon di blockchain:

Variabel Tipe Data Deskripsi Contoh Nilai Enkripsi
Transaction_ID String (Primary Key) ID unik transaksi (format: TXXX) T001 High
Conservation_ID String (Foreign Key) Referensi ke proyek konservasi C001 Medium
Block_Hash String Identifikasi unik blok blockchain 0x1a2b3c4d High
Carbon_Credits_Transferred Integer Jumlah kredit yang ditransfer 250 High
Transaction_Date Timestamp Waktu transaksi (YYYY-MM-DD HH:MM:SS) 2024-01-16 14:30:00 Medium

Contoh Data Lengkap:

Transaction_ID Conservation_ID Block_Hash Carbon_Credits_Transferred Transaction_Date
T001 C001 0x1a2b3c4d 250 2024-01-16 14:30:00
T002 C001 0x2b3c4d5e 100 2024-02-05 10:15:00
T003 C002 0x3c4d5e6f 200 2024-02-22 09:45:00

Mekanisme Transaksi:

  1. Validasi proyek oleh Conservation_Validators.csv
  2. Pencatatan di Mangrove_Conservation_Records.csv
  3. Pembuatan transaksi di blockchain
  4. Penyimpanan hash transaksi di tabel ini

Integritas Data:

Immutability

Data transaksi tidak dapat diubah setelah tercatat di blockchain

Transparansi

Semua pihak dapat memverifikasi transaksi melalui Block_Hash

Keamanan

Enkripsi tingkat High untuk data sensitif

3.3 Tabel: Conservation_Validators.csv

Mencatat validator independen untuk proyek konservasi:

Variabel Tipe Data Deskripsi Contoh Nilai Standar
Validator_ID String (Primary Key) ID unik validator (format: VXXX) V001 -
Conservation_ID String (Foreign Key) Proyek yang divalidasi C001 Harus ada di Mangrove_Conservation_Records
Validator_Name String Nama validator/lembaga Ahmad Syah Min. 3 karakter
Validation_Status Enum Status validasi (Approved/Rejected/Pending) Approved Verified Carbon Standard
Date_Validated Date Tanggal validasi 2024-01-17 Harus ≤ Date_Recorded + 30 hari

Workflow Validasi:

1. Pengajuan

Proyek dicatat di Mangrove_Conservation_Records.csv

2. Penugasan Validator

Validator independen ditugaskan oleh sistem

3. Verifikasi Lapangan

Pemeriksaan fisik dan dokumen

4. Keputusan

Status validasi dicatat di tabel ini

Contoh Query:

-- Cari proyek yang sudah tervalidasi
SELECT * FROM Conservation_Validators
WHERE Validation_Status = 'Approved'
AND Date_Validated BETWEEN '2024-01-01' AND '2024-03-31';

3.14 Tabel: Biodiversity_Monitoring.csv

Merekam hasil pemantauan biodiversitas mangrove:

Variabel Tipe Data Deskripsi Contoh Nilai Metode Pengukuran
Bio_ID String (Primary Key) ID unik pemantauan (format: BXXX) B101 -
Conservation_ID String (Foreign Key) Proyek terkait C001 Harus ada di Mangrove_Conservation_Records
Species_Count Integer Jumlah spesies flora/fauna teridentifikasi 15 Observasi langsung, kamera trap
Tree_Density Integer Kepadatan pohon per hektare 200 Plot sampling 10x10m
Water_Quality Enum Kualitas air (Good/Moderate/Poor) Good Parameter DO, BOD, pH, salinitas
Assessment_Date Date Tanggal pemantauan 2024-01-22 Min. 2x/tahun

Indikator Kesehatan Ekosistem:

Keanekaragaman Spesies

Target: Minimal 10 spesies baru dalam 5 tahun

Alat: Biodiversity_Monitoring.csv

Kepadatan Vegetasi

Target: 150-250 pohon/ha untuk zona intermediet

Formula: \(D_t = \frac{N_t}{A}\)

Protokol Pemantauan:

  1. Pemilihan plot acak (10% area proyek)
  2. Identifikasi spesies dan penghitungan
  3. Pengukuran parameter kualitas air
  4. Pencatatan hasil dalam tabel ini
  5. Analisis tren setiap 6 bulan

TIPS MANAJEMEN DATA

Best Practice Database

  • Gunakan Conservation_ID sebagai kunci relasi utama
  • Validasi data masuk dengan constraint (min/max values)
  • Backup harian untuk data kritis

Keamanan Blockchain

  • Enkripsi High untuk Transaction_ID dan Block_Hash
  • Simpan private key di hardware security module
  • Audit smart contract berkala

Validasi Efektif

  • Gunakan validator independen bersertifikat
  • Lakukan spot check 10% dari proyek
  • Catat semua hasil di Conservation_Validators.csv

Query Penting

-- Hitung total kredit karbon per proyek
SELECT Conservation_ID, SUM(Carbon_Credits_Transferred)
FROM Blockchain_Transactions
GROUP BY Conservation_ID;
